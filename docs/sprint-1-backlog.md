# Backlog tecnico priorizado - Sprint 1

## Objetivo del sprint
Establecer la base funcional segura del sistema: portal publico minimo, acceso inicial al backoffice y cimientos de seguridad/autorizacion.

## Historias priorizadas
1. Inicializar autenticacion por sesion con bloqueo por intentos fallidos.
2. Definir modelo de autorizacion base por roles (Superadmin, Administrador logial, Secretario, Tesorero, Director de estudios, Hermano, Auditor).
3. Construir layout institucional para portal publico (inicio, sobre nosotros, contacto, sobre el sistema).
4. Crear dashboard inicial de backoffice con indicadores mockeados.
5. Implementar CRUD inicial de hermanos (listado y ficha basica).
6. Crear semillas base: lodge principal, roles y usuario administrador.
